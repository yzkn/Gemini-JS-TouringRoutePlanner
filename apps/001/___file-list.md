ここまでのヒアリングと要件定義に基づき、作成・提案したソースファイルの一覧を整理しました。

これらは、GitHub Actionsによる自動更新パイプラインと、GitHub/Cloudflare Pagesによる静的配信を前提とした構成になっています。

---

### 1. データ加工・ビルド関連 (Python / CLI)
全国の道路データを大型バイク向けに加工し、ベクトルタイル（PMTiles）化するためのコアスクリプト群です。

| ファイル名                | 役割                 | 主な処理内容                                                                  |
| :------------------------ | :------------------- | :---------------------------------------------------------------------------- |
| **`process_n13_data.py`** | データ加工スクリプト | N13データの読み込み、幅員判定、勾配計算（標高タイル連携）、規制フラグの付与。 |
| **`gen_pmtiles.sh`**      | タイル生成シェル     | `tippecanoe` を使用し、加工済みGeoJSONをPMTiles形式に変換・最適化。           |
| **`regulations.json`**    | 規制データベース     | ユーザー報告やJARTICから収集した「二輪通行禁止」等の地点・リンクIDリスト。    |

### 2. フロントエンド関連 (HTML / JavaScript)
RideWithGPSスタイルの操作感を実現し、クライアント側で規制判定を行うWebアプリケーション本体です。

| ファイル名        | 役割                 | 主な処理内容                                                         |
| :---------------- | :------------------- | :------------------------------------------------------------------- |
| **`index.html`**  | アプリケーション本体 | MapLibre GL JSの描画、UIパネル（排気量切替、診断表示）の構造。       |
| **`app.js`**      | メインロジック       | OSRM連携、ウェイポイント管理、ドラッグ移動機能、経路描画。           |
| **`snapping.js`** | スマート吸着エンジン | クリック地点から「通行可能な道路」のみを抽出して吸着させるロジック。 |
| **`analyzer.js`** | 経路診断ロジック     | Turf.jsを用いた、ルート上の急勾配や規制区間のリアルタイム照合。      |

### 3. インフラ・自動化関連 (YAML)
GitHub Actionsを利用した月次更新と自動デプロイの設定ファイルです。

| ファイル名            | 役割              | 主な処理内容                                                                    |
| :-------------------- | :---------------- | :------------------------------------------------------------------------------ |
| **`data_update.yml`** | CI/CDパイプライン | 47都道府県別の並列ビルド（Matrix）、PMTilesのマージ、GitHub Pagesへのデプロイ。 |

### プロジェクト構成（ディレクトリイメージ）
これらのファイルは以下の構造で管理することを想定しています。

```text
root/
├── .github/
│   └── workflows/
│       └── data_update.yml      # 3. インフラ関連
├── data_processor/
│   ├── process_n13_data.py      # 1. データ加工
│   ├── gen_pmtiles.sh           # 1. タイル生成
│   └── regulations.json         # 1. 規制DB
├── public/                      # フロントエンド（2. 関連）
│   ├── index.html
│   ├── js/
│   │   ├── app.js
│   │   ├── snapping.js
│   │   └── analyzer.js
│   └── data/
│       └── (kanagawa_roads.pmtiles) # ビルド後に生成・配置される
└── README.md
```

これらのファイルの内容を統合・詳細化することで、実動するアプリケーションが完成します。特定のファイルのコードをさらに詳細に（例：Pythonでの具体的な標高計算ロジックなど）記述する必要はありますか？
