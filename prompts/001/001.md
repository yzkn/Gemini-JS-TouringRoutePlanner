# 要件定義：大型バイク専用ルートプランナー

## 1. システムアーキテクチャ
- **Frontend:** React / Vue.js + MapLibre GL JS（Mapbox互換のオープンソースライブラリ）
- **Routing:** Project OSRM API（経路提案）+ Turf.js（クライアント側での規制判定）
- **Data Hosting:** GitHub Pages または Cloudflare Pages
- **Tile Format:** PMTiles (HTTP Range Requestsによる単一ファイル配信)
- **CI/CD / ETL:** GitHub Actions (Matrix戦略による47都道府県並列処理)

## 2. データ処理パイプライン（GitHub Actions）

### 2.1 コミュニティ駆動の規制データ管理
- **入力ソース:** GitHubリポジトリ内の `regulations.json`
- **収集方法:**
    - アプリ上に「規制報告ボタン」を設置。
    - ユーザーが地点をクリックし、規制内容（二輪禁止、大型不可等）を入力。
    - GitHub API経由で Issue を作成、または直接 `regulations.json` へプルリクエストを投げる仕組み（簡易的にはGoogleフォーム等からJSON生成も可）。
- **反映:** 月次の定期ビルド時にこのJSONを読み込み、N13の該当リンクにフラグを付与。

### 2.2 都道府県別並列ビルド（Matrix戦略）
1.  **分散処理 (Job 1-47):**
    - 各都道府県のN13データをダウンロード。
    - **勾配計算:** 国土地理院の標高タイルからリンクの平均勾配を算出（15%以上にフラグ）。
    - **属性付与:** 幅員（N13）、規制フラグ（regulations.json）、排気量区分を整理。
    - 各都道府県単位の中間ファイル（GeoJSON/FlatGeobuf）を出力。
2.  **マージ・タイル化 (Final Job):**
    - 全47都道府県の中間ファイルを統合。
    - `tippecanoe` を使用し、PMTilesを生成。
    - **最適化（shiwaku形式）:** 低ズームでは細街路を間引き、高ズームで詳細を表示。属性は `[width, gradient, limit_type, name]` 等に極限まで短縮。

## 3. アプリケーション機能詳細

### 3.1 賢いマップマッチングと吸着
- **排気量フィルタリング:**
    - ユーザーが「125cc」を選択中：自動車専用道路属性を持つリンクへの吸着を無効化。
    - ユーザーが「大型」を選択中：すべての道路に吸着するが、規制フラグがある場所を通る際は警告を表示。
- **幅員制限の挙動:**
    - 初期状態では「幅員3.0m未満」のリンクをマッチング候補から除外。
    - 設定画面で「狭小路を許可」をONにすると、マッチング対象に含まれるようになる。

### 3.2 OSRM連携と検証ロジック
1.  OSRM APIから得られたルート（座標列）を `turf.js` の `lineSlice` 等で細分化。
2.  PMTilesから取得した属性レイヤーと重ね合わせる。
3.  **警告表示:**
    - 勾配15%以上の区間 → オレンジ色で強調。
    - 二輪通行禁止区間 → 赤色で強調 ＋ 「通行不可」ポップアップ。

### 3.3 UI/UX（RideWithGPSスタイル）
- **標高プロファイル:** 画面下部にルートの断面図を表示。勾配15%以上の箇所をグラフ上で色分け。
- **作図ツールバー:** 「自動追従（OSRM）」「直線モード」「元に戻す」「消去」を配置。
- **エクスポート:** 完成したルートを「道路属性情報付きGeoJSON」としてダウンロード。

## 4. 実装に向けた技術的アドバイス

### 4.1 標高計算の効率化
GitHub Actions内で毎回標高タイルを全件叩くと時間がかかるため、道路のノード（接点）付近の標高だけを効率よく引くロジック（または標高データのキャッシュ）が必要です。

### 4.2 PMTilesの最適化設定
`shiwaku-roadcenterline-viewer` のように、ズームレベルごとに `tippecanoe` のオプションを調整します。
- `minzoom`: 道路種別（国道はz5〜、都道府県道はz8〜、市町村道はz11〜など）で制御。
- `drop-fraction-as-needed`: データ量が多い場合に自動的に間引く。

### 4.3 コミュニティ機能の最小構成
最初は「GitHub Issueのテンプレート」へリンクを飛ばすだけでも十分です。「[通報] 都道○○号 奥多摩周遊道路：土日二輪禁止」といったIssueが立てば、それを月次ビルド時に手動または半自動でJSONにマッピングする運用から始めるのが現実的です。
